# Project
MacBook (Apple Silicon M1/M2) + Android Tablet Wired Second Display System

---

# Objective
Use an old Android tablet (Sony SGP312) as a wired secondary display for MacBook.

Primary goal:
- stable
- low latency
- wired connection
- extend display (not mirror only)

Connection method:
USB-C (Mac) <-> micro USB OTG (Tablet)

No Wi-Fi
No LAN cable
No Ethernet adapter

---

# Hardware Requirements

## Mac
- MacBook Apple Silicon
- M1 / M2
- macOS Sonoma / Ventura assumed

## Tablet
- Sony Xperia Tablet Z
- model: SGP312
- Android legacy device
- USB: micro USB (OTG capable)

## Cable
- data transfer capable
- NOT charge-only cable

Required:
USB-C ⇄ micro USB OTG data cable

Important:
must support ADB data communication

---

# Architecture

## High-level flow
Mac Virtual Display
    -> Screen Capture
    -> H.264 Encode
    -> ADB USB transport
    -> Android Decode
    -> SurfaceView render

---

# Mac-side Requirements

## 1. Virtual Display
Create logical Display 2 inside macOS.

Purpose:
macOS should recognize tablet as second workspace.

Requirement:
window drag from Display 1 to Display 2

Preferred:
Virtual display driver / software display layer

If difficult:
Phase1 = mirror MVP
Phase2 = extend display

---

## 2. Screen Capture
Capture only virtual display area.

Preferred API:
ScreenCaptureKit

Fallback:
CoreGraphics screen capture

Capture settings:
- resolution: 1280x800
- fps: 30
- pixel format: BGRA

---

## 3. Encoding
Codec:
H.264

Reason:
best compatibility for old Android hardware decode

bitrate:
4Mbps default

adaptive:
2Mbps - 8Mbps

---

## 4. USB Communication
Transport:
ADB over USB

Required command:
adb devices
adb forward tcp:9000 tcp:9000

Port:
9000

Protocol:
TCP stream over adb forward

---

# Android-side Requirements

## 1. Receiver
Language:
Kotlin

Socket receiver:
localhost:9000

Read H.264 stream continuously

---

## 2. Decode
Use:
MediaCodec

Hardware decode preferred

Fallback:
software decode if unsupported

---

## 3. Render
Use:
SurfaceView

fullscreen render

tablet orientation:
landscape default

---

# Performance Targets
- latency < 50ms
- target 20-30ms
- 30fps stable
- no frame drops during cursor movement

---

# Constraints
SGP312 is old hardware.

must optimize for:
- low CPU load
- low memory usage
- low resolution

Preferred resolution:
1280x800

Fallback:
1024x768

---

# Error Handling

## Case 1
ADB device not detected

Check:
- developer mode
- USB debugging ON
- cable supports data
- `adb devices`

---

## Case 2
black screen

Check:
- codec init
- SurfaceView lifecycle
- socket stream

---

## Case 3
high latency

Reduce:
- fps 30 -> 15
- bitrate
- resolution

---
